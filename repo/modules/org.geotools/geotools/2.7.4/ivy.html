Description

GeoTools is an open source Java library that provides tools for geospatial data.

epsg-extension Contains extra CRS defined by ESRI (and other parties) in the EPSG namespace. Those CRS are not defined in the official ESPG database, but are used widely enough to warant an optional plugin in Geotools. This plugin do not contains any definition already found in the official EPSG database - it is not a replacement for epsg-hsql or epsg-postgresql. One of the above plugins (at user choice) should be in the classpath. referencing
epsg-h2 Connection to an embedded EPSG database in H2 format. This database is built from the SQL scripts delivered by EPSG. referencing
epsg-hsql Connection to an embedded EPSG database in HSQL format. This database is built from the SQL scripts delivered by EPSG. referencing
epsg-oracle This plugin allows you to make use of an oracle database as the host of your CoordinateReferenceSystem definitions. The table names and content are provided by the EPSG database (distributed in access format) and can be loaded using a provided SQL script. referencing
epsg-postgresql Connection to an EPSG postgreSQL database. This database should be built by the user from the SQL scripts delivered by EPSG. referencing
epsg-wkt EPSG and AUTO authority factories are defined for data.crs.CRSService. These codes are used to provide CoordinateRegerenceSystem for GCE and DataStore implementations that do not define their own native definition. Contains a Property file based CoordinateSystemAuthority (the property file is too large to be included in the main geotools release). This class will need to be ported to the new GeoAPI CoordinateReferenceSystem interfaces. referencing
feature-pregeneralized Feature Source with prebuild generalizations metadata,shapefile
geojson Provides GeoJSON Encoding and Parsing support main
geometry Implementations of ISO 19107 (Spatial Schema) main
geotiff Datasource created to read GeoTIFF raster format coverage
graph Graph classes that builds and walk networks based on Feature relationships. The original graph package has been developed using the JUMP Feature API. Its intension is to be able to aid automated skeletonization of water bodies by computing directionality of generated skeletons. The work for the original JUMP based library has been funded by the BC Proventional Government department of BMGS. This library has been ported to geotools2 as part of the VWFS project sponcered by Geo Innovations. main

jp2k Extension of the of Grid Coverage plugins, leveraging on the JP2K raster data access capabilities. coverage,epsg-hsql
jts-wrapper This is a basic 2D ISO 19107 implementation which uses JTS for the computational geometry. main
main The main module contains the key interfaces and default implementations that are used by other GeoTools modules. api
matfile5 Extension of the of Grid Coverage plugins, to read matlab files with georefencing information coverage,epsg-hsql
metadata Contains implementations of metadata and some core utilities classes opengis
mysql The MySQL module contains an implementation of the DataStore interface (found in data), as well as the older DataSource interface (found in core), for use with MySQL. jdbc
opengis Standard interfaces implemented throughout the library
oracle-spatial Module offers GeoTools DataStore support for Oracle Spatial jdbc
postgis-versioned Used to extract and edit features from a PostGIS database along with feature versioning. PostGIS was developed by Refractions Research. It is an extension to the excellent Postgresql database, adding support for Geographic objects. It is open source and freely available. postgis
postgis Used to extract and edit features from a PostGIS database. PostGIS was developed by Refractions Research. It is an extension to the excellent Postgresql database, adding support for Geographic objects. It is open source and freely available. jdbc
process An API for creating custom processes/operations plus a collection of pre-written processes including vector/raster conversion coverage,shapefile,cql
property DataStore based on standard Java properties files, originally created as demo; now serves as a useful reference implementation. main
referencing Contains implementations of Coordinate Reference Systems (CRS), conversion and transformation services. metadata
referencing3D Transforms coordinates from "heights above the ellipsoid" to "heights above the geoid". This module provides the algorithms from an Earth Gravitational Model as a MathTransform implementation. Installing the JAR on the classpath should be sufficient for making vertical transformations available to an application like any other transforms. referencing
render The render module contains a renderer built around the interface in api and main. This is a first class geotools module as creating a visual representation of features is considered key to most geospatial applications. coverage,cql
sde-dummy ArcSDE dummy API
sfs SFS classes data,geojson
shapefile-renderer Renderer optimized for shapefiles shapefile,render
shapefile Shapefile module data
svg Dynamic symbolizers for SVG symbols render
swing Provides widgets for map display and other GUI elements render,wms
temporal Provides a Temporal (ISO 19108) implementation. This module contains temporal aspects implementations for geographic information. referencing,metadata
validation Validation module (and validator batch tool) for use with geotools2. This library has been ported to geotools2 as part of the VWFS project sponsered by Geo Innovations. graph
vpf VPF plugin main
wfs Implementation of DataStore allowing geotools code to work with an OGC Web Feature Service. Offers client code for WFS 1.0 and WFS 1.1. xml
wms An OGC Web Map Server client implementation that can be used directly or as through the GridCoverageExchange API. Direct usage is recommended, as the GridCoverageExchange access for WMS is not well tested and is complicated to use. It supports WMS versions 1.0.0, 1.1.0, 1.1.1, and 1.3.0. xml,render
wps Web Processing Service implementation. process
xml XML Parser based on the XDO design main
